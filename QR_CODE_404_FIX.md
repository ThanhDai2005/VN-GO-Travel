# QR CODE 404 FIX - IMPLEMENTATION COMPLETE

**Date:** 2026-05-05  
**Issue:** HTTP 404 errors when scanning QR codes  
**Root Cause:** URL path mismatch between backend and mobile app  
**Status:** ✅ FIXED

---

## 🔍 PROBLEM IDENTIFIED

### The Issue
When users scanned QR codes generated by admin web, they got HTTP 404 errors.

### Root Cause
**URL Path Mismatch:**
- Backend generated: `https://thuyetminh.netlify.app/app/scan?t={token}`
- Mobile app expected: `https://thuyetminh.netlify.app/scan?t={token}`
- Android intent filter: `DataPathPrefix = "/scan"` (no `/app` prefix)
- Result: QR opened in browser instead of app → 404 error

---

## ✅ FIXES IMPLEMENTED

### Fix #1: Backend Environment Variable
**File:** `backend/.env` (line 5)

**Changed from:**
```
SCAN_QR_URL_BASE=https://thuyetminh.netlify.app/app/scan
```

**Changed to:**
```
SCAN_QR_URL_BASE=https://thuyetminh.netlify.app/scan
```

**Impact:** QR codes now generate with correct URL format that matches mobile app intent filter.

---

### Fix #2: Static File Serving Route
**File:** `backend/src/app.js` (line 102)

**Changed from:**
```javascript
app.use('/app', express.static(path.join(process.cwd(), 'public')));
```

**Changed to:**
```javascript
app.use('/', express.static(path.join(process.cwd(), 'public')));
```

**Impact:** Web bridge HTML now accessible at `/scan.html` instead of `/app/scan.html`.

---

### Fix #3: Netlify Configuration
**File:** `netlify.toml` (NEW FILE)

**Content:**
```toml
[[redirects]]
  from = "/scan"
  to = "/scan.html"
  status = 200

[[redirects]]
  from = "/app/scan"
  to = "/scan.html"
  status = 301
```

**Impact:** 
- `/scan` serves the web bridge page (for users without app)
- `/app/scan` redirects to `/scan.html` (backward compatibility)

---

### Fix #4: Web Bridge Deployment
**Action:** Copied `backend/public/scan.html` to `public/scan.html`

**Impact:** Netlify can now serve the web bridge from project root.

## 🔄 HOW IT WORKS NOW

### Complete Flow:

```
1. ADMIN GENERATES QR CODE
   ↓
   Backend creates JWT token
   ↓
   URL: https://thuyetminh.netlify.app/scan?t={token}
   ↓
   QR code displayed in admin web

2. USER SCANS QR CODE
   ↓
   Mobile device detects URL
   ↓
   Android intent filter matches "/scan" path
   ↓
   App opens automatically (no browser)

3. MOBILE APP HANDLES DEEP LINK
   ↓
   QrResolver parses token from URL
   ↓
   PoiEntryCoordinator calls POST /api/v1/zones/scan
   ↓
   Backend returns zone + POI data

4. USER SEES ZONE POI LIST
   ↓
   POIs displayed with names, summaries
   ↓
   User can purchase zone
   ↓
   User can access POI details
```
